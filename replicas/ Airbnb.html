Airbnb (today’s MVP) — Founder Kit

Short-stay marketplace: host onboarding, search + filters, bookings, payouts, reviews.

Generate this in demo Get early access

Overview

  • Goal: Validate a two-sided short-stay marketplace in one city with tight scope and clear unit economics.
  • Primary users: Guests, Hosts, Ops Admin.
  • Platforms: Web App (guest/host), Admin web panel.
  • Payments: Stripe (Connect Express) split payments & payouts.
  • Risk controls: Host KYC (stub), phone/email verification, basic fraud checks.

MVP Scope (must-have)

  • Auth: email + magic-link; profile basics (name, phone, photo).
  • Host onboarding: list a place (title, photos, address, price, availability calendar, rules).
  • Search & filters: location, dates, guests, price range, property type.
  • Listing page (PDP): gallery, amenities, calendar, reviews, house rules, “Request to book.”
  • Booking flow: date picker, price breakdown, payment, confirmation, receipt.
  • Payouts: Stripe Connect onboarding for hosts; automatic weekly payouts.
  • Reviews: 1–5 stars + short text; basic moderation.
  • Admin (read-only): users, listings, bookings, payouts, flags.

Core Screens

  1. Home / Search — hero search, map toggle (defer map to v2), featured categories.
  2. Results — cards with photo, price, rating, quick filters (price, type, “superhost”).
  3. Listing (PDP) — gallery, host badge, amenities, calendar, reviews, booking module.
  4. Host: Create Listing — stepper: basics → photos → pricing → availability → review & publish.
  5. Checkout — guest details, payment, terms, confirmation.

Data Model (simplified)

  • User { id, role: guest|host|admin, name, email, phone, photo }
  • Listing { id, hostId, title, desc, address, coords, photos[], nightlyPrice, maxGuests, amenities[], rules[], ratingAvg }
  • Availability { listingId, date, isBlocked }
  • Booking { id, listingId, guestId, checkIn, checkOut, guests, amountTotal, status }
  • Review { id, listingId, authorId, rating, text, createdAt }
  • Payout { id, hostId, bookingId, amount, status }

API (stub)

POST /auth/magic-link
GET  /listings?city=&dates=&guests=&priceMin=&priceMax=
GET  /listings/:id
POST /bookings { listingId, checkIn, checkOut, guests, paymentToken }
POST /hosts/onboard (Stripe Connect)
POST /reviews { listingId, rating, text }

Go-to-Market

  • Wedge: niche stays (e.g., cabins near <City>), curated quality → faster liquidity.
  • Supply first: recruit 30–50 hosts in one area via local FB groups/IG/real-estate contacts.
  • Demand: SEM on “cabins near <City>”, collabs with local creators, newsletter partners.
  • Pricing: 12–15% guest fee; 3% host fee (start simple).

Risks & Mitigations

  • Regulatory zoning → start in host-friendly locales, add address checks.
  • Fraud/chargebacks → Stripe Radar, verified profiles, deposit for high-risk bookings.
  • Cold start → focus on one area; curated supply; concierge onboarding for first 50 hosts.

Next actions

  1. Publish 3–5 hero screens as a clickable demo.
  2. Onboard first 20 local listings (concierge) and run a friends-and-family beta.
  3. Measure request→booking rate and host payout satisfaction.
